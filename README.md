# 密袋鼯飼養遊戲

## 簡介

這是一款模擬飼養密袋鼯的遊戲。玩家需要照顧密袋鼯的日常需求，確保牠們健康快樂地成長。

## 遊戲玩法

- **餵食**：定期給密袋鼯提供食物和水。
- **清潔**：保持密袋鼯的生活環境乾淨。
- **互動**：與密袋鼯互動，增進感情。
- **健康管理**：注意密袋鼯的健康狀況，及時治療疾病。
- **生命週期**：密袋鼯會經歷不同的生命階段，玩家需要根據不同階段的需求進行照顧。

## 使用者故事與任務卡

📌 User Story 1：學習飼養知識

Task Cards
    •    建立「虛擬蜜袋鼯」物件（包含狀態：飢餓度、清潔度、快樂度）。
    •    設計餵食功能（選擇食物 → 狀態改變）。
    •    設計清潔功能（打掃籠子 → 清潔度提升）。
    •    設計互動功能（撫摸、遊戲 → 快樂度提升）。
    •    新手教學模式（教玩家基本操作）。
    •    建立知識提示（餵錯食物或忽略清潔時跳出提示）。

Test Cases
    •    玩家每天餵食 → 飢餓度保持正常。
    •    玩家一天不餵食 → 飢餓度下降，角色顯示不開心。
    •    玩家持續餵錯食物（例如人類零食） → 健康值下降，顯示警告。
    •    玩家清理籠子 → 清潔度提升，異味消失。
    •    玩家未清理籠子超過 3 天 → 清潔度降至危險區域，角色出現不適。
    •    玩家進行互動 → 快樂度上升，角色出現正面反應。

📌
 User Story 2：建立責任感

Task Cards
    •    建立「成長日誌」系統（記錄每日狀態變化）。
    •    設計健康值系統（根據玩家行為調整）。
    •    設計壽命/年齡參數（每天經過 = 成長一天）。
    •    設計狀態歷史圖表（飢餓、快樂、清潔變化曲線）。
    •    設定提醒通知（提醒餵食、清潔）。

Test Cases
    •    玩家持續餵養 → 成長日誌顯示穩定健康狀態。
    •    玩家 3 天不清潔 → 健康值下降並記錄在日誌中。
    •    玩家餵食過量 → 健康值異常，日誌顯示「肥胖風險」。
    •    玩家忽略照顧超過 1 週 → 角色健康急劇下降，產生失敗紀錄。
📌
 User Story 3：理解壽命與長期投入

Task Cards
    •    設計「生命週期」機制（幼年 → 青年 → 成年 → 老年）。
    •    不同年齡段行為差異（幼年更活潑，老年需要更多醫療照護）。
    •    設計長期照顧需求（例如：年紀越大，醫療需求越高）。
    •    建立「壽命縮放系統」（例如 1 天模擬 1 年）。
    •    設計紀念機制（角色死亡時，生成回顧日誌）。

Test Cases
    •    幼年階段 → 食量小，互動需求高。
    •    成年階段 → 食量大，互動需求中等。
    •    老年階段 → 容易生病，需要醫療支出。
    •    玩家全程細心照顧 → 角色活到完整壽命。
    •    玩家疏忽 → 壽命縮短，角色提前死亡。

📌
 User Story 4：模擬飼養成本

Task Cards
    •    建立「支出記錄系統」（食物、籠子、醫療等分類）。
    •    設計「消耗品」機制（食物、木屑、玩具需要定期購買）。
    •    設計「醫療事件」（隨機或因疏忽觸發 → 需支付醫療費用）。
    •    建立「財務報告」頁面（支出明細、月支出統計）。
    •    設定「預算上限警告」（超支提醒）。

Test Cases
    •    玩家購買食物 → 餘額減少，支出記錄增加。
    •    玩家忘記購買食物 → 角色挨餓，健康下降。
    •    玩家過度花費 → 系統提示「超出預算」。
    •    玩家持續良好管理 → 支出報告顯示穩定消費曲線。
    •    角色生病 → 支出記錄顯示醫療費用。

📌
 User Story 5：情感連結與互動

Task Cards
    •    設計互動模組（撫摸、聲音、遊戲）。
    •    設計 AI 反應（不同情境給不同回饋表情或動作）。
    •    建立「依賴度/感情值」系統（互動越多，角色依賴越高）。
    •    設計聲音互動（角色會辨識並回應玩家呼喚）。
    •    設計「反饋動畫/音效」（正向互動 = 高興叫聲；忽略 = 哀鳴）。

Test Cases
    •    玩家每天互動 → 角色親密度提升，顯示親近行為。
    •    玩家完全不互動 → 親密度下降，角色表現冷淡。
    •    玩家長期高互動 → 角色依賴度過高，出現「分離焦慮」提示。
    •    玩家使用聲音呼喚 → 角色做出反應（跑來、叫聲）。
    •    玩家玩遊戲 → 角色表現開心，快樂值上升。

📌
 User Story 6：飼養失敗的反饋

Task Cards
    •    設計「死亡機制」（若健康值歸零 → 角色死亡）。
    •    建立「失敗結局提示」（死亡原因總結：飢餓、疾病、忽視）。
    •    設計「警告系統」（當狀態危險時提前提醒）。
    •    建立「重新開始機制」（死亡後玩家可重新養一隻）。
    •    設計「學習建議」（失敗後提供改進建議）。

Test Cases
    •    玩家忽略餵食 → 角色健康下降，最終死亡。
    •    玩家忽略清潔 → 角色罹患疾病，導致死亡。
    •    玩家持續餵錯食物 → 中毒死亡。
    •    玩家照顧失敗 → 系統生成失敗報告與改進建議。
    •    玩家重新開始 → 可以選擇再養一隻新的角色。

# 功能需求（Functional Requirements）

## User Story 1：學習飼養知識

### FR-01 建立蜜袋鼯角色
描述：系統需生成蜜袋鼯角色，包含屬性：飢餓度、清潔度、快樂度
- 輸入：遊戲開始
- 輸出：角色生成成功
- 測試方式：
  - 角色屬性值初始正確

### FR-02 餵食功能
描述：玩家能餵食蜜袋鼯，不同食物影響不同屬性
- 輸入：玩家選擇食物
- 輸出：飢餓值增加，快樂值增加，健康值變化
- 測試方式：
  - 正確餵食 → 飢餓值增加
  - 忽略餵食 → 飢餓值下降，角色顯示不開心
  - 餵錯食物 → 健康值下降，顯示警告

### FR-03 清潔籠子
描述：玩家可清理籠子
- 輸入：點擊清潔操作
- 輸出：清潔度提升，異味消失，疾病機率降低
- 測試方式：
  - 玩家清理 → 清潔度上升
  - 未清理超過 3 天 → 清潔度危險，角色出現不適

### FR-04 互動功能
描述：玩家可與蜜袋鼯互動（撫摸、遊戲）
- 輸入：玩家選擇互動方式
- 輸出：快樂值上升，角色反應
- 測試方式：
  - 每天互動 → 角色親密度提升
  - 長期不互動 → 親密度下降
## User Story 2：建立責任感

### FR-05 成長日誌系統
描述：系統記錄每日狀態
- 輸入：每日餵食、清潔、互動
- 輸出：生成日誌與健康報告
- 測試方式：
  - 持續餵養 → 日誌顯示穩定健康
  - 忽略清潔 3 天 → 健康值下降並記錄

### FR-06 健康值系統
描述：蜜袋鼯健康值依玩家行為調整
- 輸入：餵食、清潔、互動
- 輸出：健康值變化，疾病觸發
- 測試方式：
  - 過量餵食 → 日誌顯示「肥胖風險」
  - 忽略照顧超過 1 週 → 健康急劇下降

### FR-07 壽命/年齡系統
描述：模擬蜜袋鼯每日成長，年齡影響行為
- 輸入：遊戲天數
- 輸出：角色生命階段改變
- 測試方式：
  - 幼年 → 食量小、互動需求高
  - 成年 → 食量大、互動需求中等
  - 老年 → 容易生病，需要醫療
  ## User Story 3：理解壽命與長期投入

### FR-08 生命週期機制
描述：角色經歷 幼年 → 青年 → 成年 → 老年
- 輸入：遊戲進行
- 輸出：角色生命階段及需求變化
- 測試方式：
  - 不同階段需求不同，行為差異明顯

### FR-09 長期照顧需求
描述：角色年紀越大，醫療需求越高
- 輸入：遊戲進行
- 輸出：老年階段出現醫療事件
- 測試方式：
  - 成年 → 醫療需求低
  - 老年 → 醫療需求高

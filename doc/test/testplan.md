# 測試計畫（Test Plan）

**專案名稱**：蜜袋鼯飼養遊戲  
**作者**：Lisa（測試負責）  
**版本**：v1.0  
**日期**：2025-12-05  

---

## 1. 簡介
本測試計畫針對《蜜袋鼯的一生》寵物養成遊戲之功能與品質驗證進行規劃，目的是確保遊戲功能符合需求文件（FR/NFR）、遊戲邏輯正確、使用體驗流暢，並能傳達教育目的（理解生命的重要性）。

---

## 2. 測試範圍

### 包含
- 餵食機制（FR-01）與營養/過量處理  
- 健康值與疾病機制（FR-02, FR-06）  
- 清潔與環境影響（FR-03）  
- 互動與情緒系統（FR-04, FR-05）  
- 生命週期（幼年 → 成年 → 老年）與年齡相關行為差異  
- 支出/消耗品機制與財務紀錄（User Story 4）  
- 提示/警告/教學機制（新手教學、失敗回饋）  

### 不包含（範圍外）
- 線上多人同步（若未實作）  
- 後端雲端佈署（若為單機原型）  

---

## 3. 測試目的與成功標準

### 測試目的
- 驗證遊戲核心需求（FR）與非功能需求（NFR）是否達成。
- 確保玩家在遊玩過程中能獲得正確飼養知識，避免錯誤訊息造成誤導。

### 成功標準
- 所有關鍵功能通過對應的測試案例。  
- 所有高嚴重度（P0/P1）缺陷於交付前修正。  
- 核心 NFR（易操作性、可讀性、教育性）達成。

## 5. 測試方法與測試類型
- **功能測試（Functional）**：逐條驗證 FR。  
- **邊界/例外測試（Boundary / Negative）**：如飢餓值 0 或 100、過量餵食。  
- **整合測試（Integration）**：驗證餵食 → 健康 → 情緒 → 事件的串接。  
- **回歸測試（Regression）**：每次修正後驗證核心流程。  
- **使用者驗收測試（UAT）**：非開發者實測，回饋教育性與可用性。  
- **探索性測試（Exploratory）**：發現未列於 Test Case 的錯誤。

## 11. 範例測試案例（Sample Test Cases）

> 每一個 Test Case 應包含：編號、標題、對應需求、進入條件、步驟、預期結果、實際結果欄位（測試時填寫）

---

### **TC-01：餵食成功（FR-01）**

- **進入條件**：飢餓值 ≥ 60  
- **測試步驟：**  
  1. 開啟遊戲  
  2. 點選「餵食」→ 選擇安全食物  
  3. 確認餵食動畫或提示  
- **預期結果：**  
  - 飢餓值下降（如 -20）  
  - 情緒值提升  
  - 未出現負面健康效果  

---

### **TC-02：餵錯食物（負面影響）**

- **進入條件**：不限  
- **步驟：**  
  1. 選擇不可餵食之食物（如人類零食）  
- **預期結果：**  
  - 系統阻止餵食並顯示警告  
  - 健康值下降或觸發中毒狀態（若設計包含）  

---

### **TC-05：未清潔影響健康（FR-03）**

- **進入條件**：清潔度 < 30%  
- **步驟：**  
  1. 模擬多日未清潔  
- **預期結果：**  
  - 健康值下降  
  - 可能出現生病狀態與警告提示  

---

### **TC-11：死亡機制（FR-06）**

- **進入條件**：健康值長期偏低  
- **步驟：**  
  1. 模擬或設定健康值 = 0  
- **預期結果：**  
  - 顯示死亡動畫或事件  
  - 顯示原因總結  
  - 允許重新開始  

---

## 12. 測試報告與紀錄方式
- 每次測試需紀錄：Test Case 編號、時間、測試者、實際結果、是否通過、缺陷編號  
- 缺陷使用 GitHub Issues 管理，標記 severity / priority  
- 修正後需進行回歸測試  

---

## 13. 缺陷分類與優先級
- **P0（阻斷）**：遊戲無法啟動、主要流程崩潰  
- **P1（高）**：主要功能錯誤（餵食失效、健康計算錯誤）  
- **P2（中）**：提示不清、UI 錯字  
- **P3（低）**：美術顯示錯誤、文案微調  

---

## 14. 測試完成標準
- 所有 P0/P1 缺陷皆修復  
- 所有核心 Test Case（TC-01~TC-11）皆完成並通過  
- 測試負責人（陳晶晶）完成最終測試報告並簽核
